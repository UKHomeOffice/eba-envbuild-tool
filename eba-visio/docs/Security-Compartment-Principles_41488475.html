Security Compartment Principles

The following principals are used when placing hosts within different security compartments in the EBSA physical designs:

  • The aim is to have the least number of compartments within a virtual machine container
  • When an application that talks to another application, those applications will be placed in different security compartments
  • When an application that talks to the database, they will be placed within different security compartments
  • When an application has a user interface that is separate to the application then they will be placed within different security compartments
  • When databases do not interact with each other they should be placed within the same security compartment
  • When there is transfer of business data where no processing has been done they can remain in the same security compartment
