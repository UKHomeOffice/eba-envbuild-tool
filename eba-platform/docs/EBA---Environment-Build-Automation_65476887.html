EBA - Environment Build Automation

Introduction

A set of environment build tools has been developed as part of the Home Office IPT programme. This project delivers these tools in a re-usable way, showing how they have been used together on the IPT programme to facilitate the provision of thousands of VM's across many environments, with tooling support for project teams to deploy their own applications to their allotted environments.

The IPT programme environment build tools have been packaged into 4 projects and released to the Home Office Github account. Two of those packages are fully open sourced and available to the public and two of them are available on an invitational basis only.

  • EBA - Visio (Open sourced)
  • EBA - ACloud (Open sourced)
  • EBA - Self Service
  • EBA - Environment Build

The footprint of these projects onto the overall design is shown in the diagram below. 

Note that the open sourced ACloud and Visio have been made to work together (see EBA - OS - Landing Page) in the absence of full Environment Build framework for the general public; if you are setting up a small number of machines this is fine. However if you need thousands of machines and years of history, the Environment Build framework (consisting of the "EBA - Environment Build" yellow box above, plus the "EBA - ACloud" and "EBA - Visio" components in red) is useful to aid maintainability.

User Guides

The following set of guides have been assembled to show how the EBA tools can be used to design and provision cloud based environments as well as deploy configured applications across those environments.

  • EBA - Visio Physical Design - How to use the Visio Physical Design tool to create physical network designs that can be used by the Environment Build tool.
  • EBA - Detailed Jenkins Environment Build flow - How to use the Environment Build tool to provision environments in the cloud.
    • EBA - Build Plans - How to create the overall build plan that the Environment Build tool requires to build your environments.
  • EBA - Deployment Self Service User guides - How to use Self Service to deploy applications to environments.

Standards and useful links

This is collection of standards that the EBA tools are known to work with. Care should be taken when deviating from these standards as this could cause the EBA tool chain to not work as expected. For example the Host Naming Standard (see EBA - Infrastructure Naming Standards) is used as a source of facts when working with Puppet and Hiera data and 'role' is taken as the first 3 characters of the hostname.

  • EBA - Infrastructure Naming Standards - The infrastructure naming standards are important, besides just being a useful guide, as following them supports a source of "facts" that can be adopted when authoring puppet data.
