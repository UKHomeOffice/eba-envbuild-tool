EBA - Glossary

Term:

Meaning:

Action PlanA sequence of one-to-many individual execution plans or jobs generated from the Build Plan in the Jenkins Build Environment instance. This is logically identical to the Build Plan, except it's in the format of actual SSH commands and individual executable Execution plans that can be sent to ACloud, instead of the Build Plan's XML format.
APIApplication Programming Interface - A set of routines, protocols and tools for interacting with a software component programmatically.
Build PlanA master XML file with logical definitions of VM's and networks plus sequencing information to describe what order machines should be built in. Includes Puppet and script execution over SSH, does not include middleware configuration. Used by the Jenkins Build Environment instance. 
CISCentre for Internet Security (http://www.cisecurity.org/) provides a set of security benchmarks that originate from consensus-oriented information security products, services, tools, metrics, suggestions, and recommendations as a public service to Internet users worldwide
CITComponent Integration Test - Can also be an environment used to test the interactions and interfaces between components.
EBSAEnvironment, Build, Support & Administration was responsible for implementing and managing all aspects of the IPT programme environments and delivery toolset including low-level design.

Environment

 

A set of virtual applications. See EBA - Infrastructure Naming Standards.Typical Environment shortnames and values are;

  • DEV - Development
  • CIT - Component Integration Test
  • SST - Supplier System Test
  • SIT - System Integration Test
  • TRN - Training
  • PRF - Performance
  • PRP - Pre-production
  • DMR - Data Migration Rehearsal
  • PRD - Production
  • TLG - Tooling (Management Tooling, Test Tooling and Development Tooling)

Environment container

 

A top-level set of environments and virtual applications (see below). See EBA - Infrastructure Naming Standards. Typical Environment container shortnames and values are;
  • NP - Non-Production environments
  • NPA - Non-Production environments hosted in AWS
  • PR - Production environments
Environment Container FileA network that joins multiple VMCs (Virtual Machine Containers) together. It typically represents an organisation's backbone network that changes rarely.
Environment Definition FileAn XML file that represents details of VM's, application networks (as used to transfer data between applications in the same VMC), NIC's, firewalls, security compartments and VMCs.
Execution PlanAn XML file with specific instructions on how to execute a set of instructions on a tightly-defined class of VM's. Created from the Build Plan by the Jenkins Build Environment instance.
FQDN

Fully Qualified Domain Name is the full name of a server. In EBSA, it's as the below;

IPInternet Protocol address - A numeric address assigned to each network connection interface on a virtual machine.
IPTImmigration Platform Technologies - Home Office programme to provide immigration platform technologies
MTZOManagement Tooling Zone, a network layer
NP Non-production environment
NPANon-production Amazon environment
POMProject Object Model - Build file used by Maven
PRProduction Environment
REBRepeatable Environment Build - basically the Jenkins Build Environment setup.
RHELRed Hat Enterprise Linux - A Linux operating system distributed by Red Hat
RPMRedhat Package Manager - Standard format for Redhat and CentOS installation packages.
SSHSecure Shell - A secure data communication protocol used to access the shell account on Linux operating systems
STSystem Test environment
VMVirtual Machine - Software based emulation of a computer
WLSTWebLogic Scripting Tool - Scripting language used to manage WebLogic
XMLeXtended Markup Language - A standardised method of defining file formats, including the allowed data items and data types within them.
YAMLYAML Ain't Markup Language - A human-readable data serialization language
YUMYellowDog Update Manager - System software that enables a repository of RPMs to be deployed.
Virtual Application

A set of applications within an environment with a defined purpose. See EBA - Infrastructure Naming Standards. Sample values
include;

  • CIZ<nr> - Continual Integration Zone
  • COBC - Core Boundary Control
  • CTZO - Collaborative Tooling Zone
  • DABC - Data Boundary Control
  • DAZO - Data Zone
  • DEZ<nr> - Development Zone
  • DTZO - Development Tooling Zone
  • EMZO - Environment Management Tooling Zone
  • ESZO - External Simulator Zone
  • INBC - Internet Boundary Control
  • PRZO - Private Application Zone
  • MABC - Management Boundary Control
  • MEBC - Management Edge Boundary Control
  • MTZO - Management Tooling Zone
  • SSZ<nr> - System Test Zone
  • TTZO - Test Tooling Zone
