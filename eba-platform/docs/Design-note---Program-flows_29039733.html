Design note - Program flows

Input

  • Location where the build plans are - GIT repo / directory
  • Environment to be built
  • VApp or set of VApps within that environment to build.

Build plans

Organisation

The program will read all the xml files recusively within a designated folder and the sum of all the files will be the model which the program uses.  e.g.

  • environmentdefinitions
    • common.xml
      • stuff
        • environments.xml - Contains the list of environments and the references to the build plans that they hold
        • plan_<name>.xml - Contains a specific build plan with a specific name
      • otherstuff/
        • something.xml
