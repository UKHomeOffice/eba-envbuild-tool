EBA - Agnostic Cloud Schema

The following diagrams represent the topologies associated with Skyscape and Amazon Web Services. In order to be able to design networks in a common way that can be provisioned in either of these cloud providers a provider agnostic topology, implemented as an XML schema, has been created for the EBA - Environment Build Automation tooling.

 

 

Skyscape VMWare

Amazon Web Services

  • There is no concept of a region, it is implicit.
  • A virtual data center (vDc) concept in vmware relates to a set of infrastructure resources that define memory, cpu, storage and bandwidth. These resources are made available to vApps to consume. You can choose more expensive/redundant/performant resources based on the level of service required. An organisation and a vDc can have more than 1 vDc.
  • A vShield Edge (vSe) provides a network security solution for vDc's. It can provide security features such as firewalling, VPN, NAT, DHCP and web load balancing capabilities.
  • A vApp is a collection of VM's that are isolated, they are typically used for host an application tier.
  • An Org network is used to provide access to other vApps or to allow access to data from external sources.

 

 

  • A region is a geographical location where data centres are available.
  • An availability zone is an actual data centre that can host infrastructure resources. 
  • A VPC is used to launch resources for your virtual network of instances. It can span multiple availability zones and be used to organise these resources. This is similar to an Organisation in VMware in terms of how the virtual infrastructure is ring fenced.
  • The VPC takes on responsibility for gateway access and holding the security features that are provided by VMware's VShieldEdge.
  • A VPC logically addresses the subnets for the virtual network. These subnets are similar to the concept of a vApp in VMware. The can be hosted in separate accessibility zones so different tiers of an application can be hosted in different data centres. 
  • An instance is logically connected to a subnet, a VM in VMware is equivalent to an Amazon Instance.
  • The concept of a vApp Network is redundant in the Amazon model as it is covered by a Subnet.
  • Amazon provide load balancing through an additional feature called Elastic Load Balancing which can be configured to automatically distribute traffic to Instances. This can be attached to VPC, but is an additional feature.
